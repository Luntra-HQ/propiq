## Data Gaps & Tracking Recommendations

### ❌ What We CANNOT Track (Critical Gaps)

#### 1. Pricing Page Views
**Impact:** HIGH
**Why it matters:** Can't identify users who are considering upgrade
**How to fix:**
```javascript
// Add to pricing page
analytics.track('Pricing Page Viewed', {
  userId: user.id,
  timestamp: Date.now(),
  referrer: document.referrer
});
```

**Convex schema addition needed:**
```typescript
pageViews: defineTable({
  userId: v.id("users"),
  pageUrl: v.string(),
  pageName: v.string(), // "pricing", "dashboard", "calculator"
  timestamp: v.number(),
  sessionId: v.string(),
})
  .index("by_user", ["userId"])
  .index("by_page", ["pageName"])
```

#### 2. Upgrade Button Clicks
**Impact:** HIGH
**Why it matters:** Can't identify abandoned upgrade attempts
**How to fix:**
```javascript
// Add to upgrade buttons
onClick={() => {
  analytics.track('Upgrade Button Clicked', {
    userId: user.id,
    plan: 'starter', // or 'pro', 'elite'
    source: 'dashboard-banner', // or 'pricing-page', 'limit-modal'
    timestamp: Date.now()
  });
  // Then redirect to Stripe...
}}
```

**Convex schema addition needed:**
```typescript
userActions: defineTable({
  userId: v.id("users"),
  actionType: v.string(), // "upgrade_clicked", "calculator_used", "analysis_shared"
  actionContext: v.string(), // JSON with details
  timestamp: v.number(),
})
  .index("by_user", ["userId"])
  .index("by_action", ["actionType"])
```

#### 3. Calculator Usage
**Impact:** MEDIUM
**Why it matters:** Calculator is a key feature - want to know who uses it
**How to fix:**
```javascript
// Track when user opens calculator
analytics.track('Calculator Opened', {
  userId: user.id,
  calculatorType: 'deal-calculator', // or 'roi-calculator'
  timestamp: Date.now()
});

// Track when user runs calculation
analytics.track('Calculator Used', {
  userId: user.id,
  inputs: { purchasePrice: 500000, downPayment: 100000 }, // anonymized
  timestamp: Date.now()
});
```

#### 4. Email Open/Click Rates
**Impact:** MEDIUM
**Why it matters:** Can't measure email effectiveness
**Current status:** emailLogs table exists but not populated
**How to fix:** Integrate with Resend webhooks or email tracking service

#### 5. Feature Usage Tracking
**Impact:** MEDIUM
**Why it matters:** Don't know which features drive engagement
**How to fix:** Track all major feature interactions:
- Property search
- Analysis export
- Report sharing
- Scenario comparisons

**Impact:** LOW-MEDIUM
